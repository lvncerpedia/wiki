## 未来予測：「PRというフォーマット自体が変わる」

まず面白い逆説があって

```
AIがコードを書くほど → AIレビューが必要になる

理由：AI生成コードは人間より脆弱性が15-18%多い（Opsera 2026）
　　　かつ、AIのPRは人間より4.6倍レビュー待ちが長い（同レポート）
```

なので**ボトルネックがコーディングからレビューに移動した**のが今。

将来像はこんな感じで見てる：

```
【今】  書く(AI) → レビュー(人間+AI) → マージ
【近未来】書く(AI) → 低リスクは自動approve / 高リスクだけ人間
【遠未来】仕様書く(人間) → 実装+レビュー全部AI → 人間は意図確認だけ
```

**PRというフォーマット自体が「意図確認の場」に変質する**と思ってる。コードの良し悪しよりも「これは本当にやりたいことか」を問うフェーズに。
