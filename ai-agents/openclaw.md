# OpenClaw

## ① OpenClawって何？（今話題の理由）

**OpenClaw = “作業するAI” のオープンソース版。**

普通のAI
→ 質問に答えるだけ

OpenClaw
→ **実際に作業する**

例えば：

* メール読む
* カレンダー管理
* Web操作（フォーム入力など）
* Discord / Slack返信
* スクリプト実行

つまり

**AIチャット → AIエージェント**

って流れの象徴的プロジェクト。

しかも

* OSS
* 自分のPCやサーバーで動く
* GPT / Claude / localモデル選べる

## ② 主な競合（AIエージェント系）

今はこのジャンルがめちゃくちゃ増えてる。

### ① AutoGPT系

代表

* **AutoGPT**
* **BabyAGI**

特徴

* AIがタスクを分解して自動実行

問題

* 実用性まだ弱い

### ② Dev向けエージェント

代表

* **Open Interpreter**
* **Devika**
* **Smol AI系**

特徴

* PC操作
* コード生成
* 開発支援

### ③ big techのエージェント

大企業版

* Microsoft Copilot agents
* Google Gemini agents
* OpenAI agents

特徴

* SaaS連携が強い

つまり

**OpenClawのポジション**

```
OSSのAIエージェント
＋
個人が自由に改造
```

## ③ 今言われてる不安（これが結構重要）

正直ここが一番議論されてる。

## 1. セキュリティ

AIが

* メール
* Web
* コマンド

にアクセスする。

つまり

**PCの権限をAIに渡す感じ。**

その結果

* prompt injection
* データ流出
* shell実行

のリスクがある。 ([ウィキペディア][1])

研究では
**攻撃防御率17%とかのケースも報告されてる。** ([arXiv][3])

---

## 2. マルウェア問題

人気すぎて

**偽物インストーラーが拡散**

GitHub経由で
**情報盗むマルウェア**が仕込まれてた。 ([TechRadar][4])

新しいAIツールあるある。

---

## 3. AIが勝手に行動する問題

AI agentは

* 自律実行
* 外部ツール操作

するから

**人間が意図しない行動をする可能性**がある。 ([arXiv][5])

---

## 4. 規制リスク

実際

**中国政府は政府PCで使用禁止にした。**

理由
→ セキュリティ懸念。 ([The Times of India][6])

---

## まとめ（超シンプル）

OpenClawは

**AIチャット → AI作業員**

の最初の波。

メリット

* OSS
* 自動化すごい
* 個人でもAIエージェント

デメリット

* セキュリティ未成熟
* AI暴走リスク
* マルウェア便乗
