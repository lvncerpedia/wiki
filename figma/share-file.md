## Figmaファイルの共有方法

**`.fig`ファイルとしてエクスポート**できるよ！
- メニュー → `File > Save local copy...` でダウンロード
- 受け取った人が `Import` すれば同じデザインが再現される ✅

ただし**注意点**があって：
- フォントが相手の環境にないと崩れる
- チームライブラリへの参照は切れる場合がある
- バージョン履歴は引き継がれない

---

## 「コード上で定義したい」について

これが本命の質問だよね？いくつか選択肢があるよ。

### 1. **Figma Config (variables)** をJSONエクスポート
デザイントークン（色・タイポグラフィ等）をコードで管理できる

### 2. **Figma REST API** でプログラム的に取得
```js
GET https://api.figma.com/v1/files/:file_key
```

### 3. **コードベースでデザインを定義したいなら → Storybook / Style Dictionary**
デザイントークンをコードで管理して、Figmaと同期させるアプローチ

### 4. **Figma as Code ツール**
- `figma-api` npm パッケージ
- `token-transformer` でデザイントークンをCSS/JSに変換

## ケース①：チームで変更を追いたい

**→ Figma + Git連携（トークン管理）がベスト**

```
Figma Variables/Styles
       ↓ (自動同期)
JSONファイル → Git管理 → PRでレビュー
```

**ツール構成：**
- **Figma Tokens Plugin** or **Variables REST API** でJSONエクスポート
- それをGitにコミット
- 変更差分がPRで見える 👀

**デメリット：** デザイン「構造」（コンポーネントの配置とか）まで追うのは難しくて、**トークン・変数レベル**が主戦場になる

---

## ケース②：自分→他人への一方的な共有

**→ `.fig`ファイルをGit管理がシンプル**

```bash
git add design.fig
git commit -m "v1.2 ボタンデザイン更新"
git push
```

相手は最新の `.fig` をダウンロードしてインポートするだけ。

**デメリット：** バイナリファイルだから差分は見えない（変更履歴はコミットメッセージ頼み）

---

## 正直なところ

| | チーム追跡 | 一方的共有 |
|---|---|---|
| `.fig` + Git | ❌ 差分見えない | ✅ シンプル |
| トークンJSON + Git | ✅ 差分見える | △ セットアップ必要 |
| Figma標準の履歴機能 | ✅ 一番楽 | ✅ URL共有だけ |

**ぶっちゃけ**、Figmaには標準でバージョン履歴があるから、**Gitでやりたい理由**が何かによって変わってくるんだよね。

「エンジニアと同じリポジトリで管理したい」とか「CIに組み込みたい」とか、そういう背景ある？🤔

## `.fig`ファイルの作り方

ファイルを開いて、左上の **Main menu → File → Save local copy** を選ぶだけ！

それだけで `.fig` ファイルがダウンロードされるよ。

---

## インポートする側は？

Figmaデスクトップアプリを開いて、`.fig`ファイルをそのままドラッグ＆ドロップすれば OK！Draftsに追加されるから、そこから好きなプロジェクトに移動できるよ。

---

## ⚠️ 注意点

**editアクセス権がないとSave local copyがグレーアウトされて使えない**から気をつけて。viewアクセスしかない場合は、ファイルオーナーが明示的に許可してないと無理。

あと`.fig`は独自フォーマットで将来変わる可能性があるって公式が言ってるから、サードパーティツールに渡すのはあんまり推奨されてないみたい。
