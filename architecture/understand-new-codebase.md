100くらいなら全然ちょうどいい規模だね。しかもフロントだけに絞るなら、ちゃんと整理すれば数日で「全体像思い出した」って状態まで戻れると思うよ。

で、この条件ならドキュメントの切り方はかなりシンプルにできる。

---

## 結論：フロントは「画面単位」でまとめるのが一番いい

バックエンドは機能単位が合うけど、フロントはこっちの方が自然：

* ログイン画面
* 一覧画面
* 詳細画面
* 作成フォーム

っていう「ユーザーが触る単位」。

理由はシンプルで、フロントは最終的に
**「どの画面が何してるか」** が分かればほぼ勝ちだから。

ここが一番大事。リハビリ用の地図。

  * Next.js
  * React
  * 状態管理（あれば）
* ディレクトリ構造

例：

```
- components が共通UI
- lib/api がバックエンド通信
```

これあるだけで復帰速度かなり変わる。

## ドキュメントとの相性めっちゃいい

さっき話してたmdにこう貼れる：

```
スマホ用メイン表示ロジック
https://github.com/.../SmartPhonePage.tsx#L325-L327
```

これやると、

* コードコピーしなくていい
* 最新状態を参照できる
* 差分が出てもズレない

かなり賢いやり方だと思う。
