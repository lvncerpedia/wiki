## 背景：なぜ今 "脱GitHub" なのか

まず前提として、GitHubが**ガチでやばいことになってる。
2025年5月〜2026年4月の間に**257件のインシデント**が記録されて、うち48件が重大障害。2026年2月は**37件**でワーストの月だった。
GitHub CTO の Vlad Fedorov が「2025年10月に10倍のキャパシティ増強計画を始めたが、2026年2月には**30倍**規模が必要だと気づいた」と公式ブログで認めてる。

**原因はAIエージェント**↓

AI コーディングツールが 2025 年初頭から使用量を爆発させ、GitHub は 2025 年 10 月に 10 倍のキャパ増強を計画したが、それでも追いつかなかった。
実際、GitHub の Merge Queue バグで292個のプルリクが消滅し、HashiCorp 共同創業者でありGitHubユーザー **#1,299 番** の Mitchell Hashimoto が、50K スターの Ghostty プロジェクトを移行させた。

```
GitHub が壊れた理由の構図

  人間1人  →  1日数commit →  従来のGitHubで余裕
  AIエージェント →  秒間数百commit →  設計限界を超えた 💥
```

## 新プレイヤー3つ（全部ここ数週間の話！）

### 1. Cursor Origin（2026/6/16 発表、激アツ）

Cursor が **2026年6月16日**に発表した Git ホスティング + レビュー + コラボプラットフォーム。
「エージェント時代のための GitHub 代替」として位置づけられてる。
現在はウェイトリストのみで、GA は 2026年秋の予定。

デモでは **1リポジトリで秒間22.6コミット**を実現。
S3 をソースオブトゥルースにして、NVMe の Git ファイルサーバーとインフィニット・レプリカで世界同期してる。

Cursor は 2023年にVSCodeフォークとして始まり、2025年12月にコードレビュースタートアップ「Graphite」を買収。
Originで「書く・ホストする・レビューする」を全部抑えようとしてる。

> **懸念点**：「コードをCursorで書いて、Originでホストして、それが全部フロンティアAIモデルの学習データになる」というトレインはかなり怖い、という意見も出てる。

ちなみにこれ書いてる間にも SpaceX が xAI 経由で Cursor を買収 するというニュースまで出てて、もう訳わからん状態になってるw

### 2. Zed DeltaDB（2026/6/11 発表）

Zed Industries が6月11日にウェイトリストを公開。ベータは「数週間後（6月末〜7月）」の予定。
コンセプトが一番ユニークで面白い↓
Git がコミット単位でスナップショットを撮るのに対し、DeltaDB は「コミットとコミットの間」＝AIとの会話や編集操作まるごとを記録する。

```
Git の世界観
  [commit A] ----→ [commit B] ----→ [commit C]
  ※間の会話や試行錯誤は消える

DeltaDB の世界観
  [会話]→[edit]→[edit]→[会話]→[edit]→[edit]→[commit]→...
  ※全部トレースできる！キャラクターレベルで追跡
```

CRDT（Conflict-free Replicated Data Type）ベースで、複数の人間とAIエージェントが同じファイルを同時編集できる設計。
Sequoia Capital リードで Series B $3,200万調達済み。

### 3. GitLab の反撃キャンペーン

Originや DeltaDB みたいな「新設計」じゃなくて、GitLabは**既存プラットフォームとして真っ向勝負**に出た。
GitLab が `about.gitlab.com/switch` というページを作って、2018年の GitHub 買収時から今まで開発者が言ってきた不満の声をまとめて「あなたたちの予言は当たった」と煽る。
主な武器は **DevSecOps オールインワン**（CI/CD + セキュリティ + AI）で、移行支援も積極的にやってる。

## 3者の比較

| | Cursor Origin | Zed DeltaDB | GitLab |
|---|---|---|---|
| 発表 | 2026/6/16 | 2026/6/11 | 既存（攻勢強化） |
| 状態 | ウェイトリスト | ウェイトリスト | 今すぐ使える |
| VCS | Git互換 | Git+独自(CRDT) | Git |
| 設計思想 | エージェントファースト | コミット間も記録 | DevSecOps一体型 |
| 懸念 | SpaceX傘下・学習利用? | 作業中コードのプライバシー | GitHubより知名度低い |
| GA予定 | 2026年秋 | 2026年夏 | 即日 |

## まとめると

「AIカンパニーたちは、GitHubがAI企業になるより早く、GitHubになろうとしている」—— というのが今の構図。
GitHubは**人間のペース**で設計されたプラットフォームで、AIエージェントの**機械のペース**についていけなくなってる。
その隙を Cursor・Zed・GitLab の三者が全然違うアプローチで狙ってる感じ。
一番ドラマチックなのは間違いなく Cursor Origin だけど、個人的には Zed の DeltaDB の思想が一番面白いと思うな〜！
